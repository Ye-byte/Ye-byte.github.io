全平台的个人密码管理器-Bitwarden
发表于2020-03-02|更新于2020-08-09|learning服务器
|字数总计:974|阅读时长:3分钟|阅读量:

1.密码保护的重要性

现在，随着网络便捷性的发展，几乎每进去一个网站便注册一次，记录密码一次，为了方便，我们常常会把很多网站的密码设置成相同或者类似的密码，这样也就隐藏了不安全的因素，容易被人破解，造成无法挽回的损失。

也正如此，催生许多提供密码管理的公司，如1password，LastPass等厂商，但是这些公司都是收费的，并且有过泄露的事件，所以把密码放进这些公司保护也并不是明智自选。

那既然不安全，我们就要搭建自己的密码管理器了，开源的密码管理器有几个KeePass，Enpass、SafeInCloud等，但是我选择了Bitwarden，因为可以把管理器搭建在自己的服务器上，完全掌握，而且支持全平台！！！这个是我的刚需！

2.Bitwarden是什么？

1.解决密码管理问题

个人、团队和业务组织存储、共享和同步敏感数据的最简单、最安全的方式。

2.开源的解决方案

该项目在Github上全部开源,担心有后台可以去查看源代码

3.全平台的管理器

无论是ios，macos，windows，Android，chrome，都有相应的软件使用

3.使用宝塔安装bitwarden

本文使用bitwarden_rs,非官方镜像，比较轻量，GitHub开源

1.准备工作

  • VPS (本文 VPS 系统为 Centos7.6)
  • 一个域名且解析到了 VPS 的 IP 地址

2.安装宝塔

使用 SSH 连接工具，如宝塔远程桌面助手连接到您的 Linux 服务器后，挂载磁盘，根据系统执行相应命令开始安装（大约2分钟完成面板安装）：

中间按y完成安装

安装成功后出现登录地址及账户密码：

3.安装Bitwarden

1.进入宝塔后台，安装推荐的环境，等待安装成功后，进入软件商店管理安装Docker：

打开Docker管理器的镜像管理功能，获取Bitwarden_rs的官方镜像bitwardenrs/server:latest：

提示Pull成功即可。

2.获取镜像后，在容器列表中创建一个容器，容器端口为80，服务端口写一个未使用的端口，这里用56560代替，然后点击+创建端口映射。服务器目录填上/docker/bitwarden/data（根据你建的文件夹的路径修改），容器目录为/data,点击+创建目录映射。完成后点击提交。

3.添加一个站点，写上解析域名，数据库可以不创建，PHP版本选个伪静态就行。然后提交。

4.打开你刚刚添加的网站设置页面，点击SSL，使用宝塔的SSL证书申请功能为刚刚的域名申请一个证书，然后部署，同时开启强制HTTPS功能，接着点击方向代理，添加一个反向代理。代理名称随意，目标URL为http://127.0.0.1:6666(刚刚设置的那个服务端口)点击提交。

5.打开之前输入的网址，即可访问自建好的密码管理器：

4.结语

到此，个人密码管理器已经搭建好了，但是目前处于开放注册状态，每个人都可以注册，后期出文章说明如何修改注册权限，使密码管理器只属于自己。
